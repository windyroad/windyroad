## 4. Blocking findings

- **AI-REPRO-01-B01 — No executable end-to-end subscription collection runner exists.** `subscription.mjs` only validates supplied status strings, renders text, and constructs command arrays (`research/llm-review-sequences/subscription.mjs:4-52`). Repository-wide callers are tests only (`research/llm-review-sequences/subscription.test.mjs:11-66`). `collectCall` accepts an abstract `send` function and API-style HTTP/provider/endpoint records (`research/llm-review-sequences/collection.mjs:104-229`), while its command-line entry point only generates the superseded API-design ledger (`research/llm-review-sequences/collection.mjs:390-400`). The active ecological entry point likewise generates artifacts and a schedule only (`research/llm-review-sequences/ecological.mjs:135-150`). Consequently, the manuscript’s claims that a runner starts fresh processes, checks auth and versions, parses product output, records provenance and attempts, detects tool use, pauses for subscription resets, and suspends on drift are not implemented or demonstrated (`research/llm-review-sequences/paper/paper.tex:168-215`). Attempt records are returned in memory but never durably written (`research/llm-review-sequences/collection.mjs:322-356`). Before collection, implement and test the minimal end-to-end runner, including crash-safe/resumable ledgers and non-prompt preflight.

- **AI-REPRO-01-B02 — The configured executable does not resolve to the frozen Codex client in the reviewed environment.** The manifest freezes Codex CLI `0.137.0` (`research/llm-review-sequences/study.json:28-34`), but `buildSubscriptionCommand` invokes unqualified `codex` (`research/llm-review-sequences/subscription.mjs:30-39`). Here that resolves first to a broken `0.101.0` installation missing its platform package; therefore both `codex --version` and `codex login status` fail. A separate `0.137.0` binary is correctly authenticated, but the constructed command would not use it. Pin or preflight the executable resolved by the runner and abort unless that exact executable reports version `0.137.0` and ChatGPT authentication.

- **AI-REPRO-01-B03 — Freeze and collection-authorization state is contradictory.** The manifest says `"frozen": true` (`research/llm-review-sequences/study.json:3-6`) and `"subscription_calls_authorized": true` (`research/llm-review-sequences/study.json:332-355`), while the active design is still pending internal review and OSF registration (`research/llm-review-sequences/study.json:21-25`), the preregistration prohibits collection before those gates (`research/llm-review-sequences/preregistration-v2.md:1-3`), and the review protocol requires resolved reviews, verified authentication, and a recorded OSF identifier before freeze (`research/llm-review-sequences/independent-review.md:128-137`). Mark the active v2 candidate and subscription calls unambiguously unfrozen/unauthorized until every gate is satisfied.

## 5. Major findings

- **AI-REPRO-01-M01 — Clean-checkout instructions are not self-contained or concurrency-safe.** They omit `npm ci`, so the literal clean-checkout test command fails before testing. They also prescribe shared global `/tmp` directories (`research/llm-review-sequences/independent-review.md:24-34`; `research/llm-review-sequences/README.md:31-48`). This produced a real cleanup collision during concurrent internal review, and both generators reject nonempty output directories (`research/llm-review-sequences/ecological.mjs:14-17`; `research/llm-review-sequences/collection.mjs:22-25`). Document the Node/npm prerequisites and `npm ci`, and use per-run `mktemp` directories.

- **AI-REPRO-01-M02 — Active and superseded collection configurations remain operationally interleaved.** Although `study.json` labels the following fields legacy (`research/llm-review-sequences/study.json:88`), the standalone `collection.mjs` command reads the top-level paid-API `models` and `preregistration_v2_candidate` and generates the 115,200-boundary legacy ledger (`research/llm-review-sequences/collection.mjs:390-400`). The active subscription schedule is generated through a different path (`research/llm-review-sequences/ecological.mjs:141-148`). Namespace the legacy manifest or require an explicit legacy flag so the obvious collection entry point cannot select the superseded route.

- **AI-REPRO-01-M03 — Subscription rate-limit behavior is inconsistent with the preregistration.** The preregistration requires pausing until the same subscription window resets and retrying after the applicable reset (`research/llm-review-sequences/preregistration-v2.md:89-95`). The existing collector instead interprets API-like HTTP status codes, sleeps 2 or 8 seconds or at most 60 seconds, and suspends for longer retry-after values (`research/llm-review-sequences/collection.mjs:137-157`). The actual CLI rate-limit output is not parsed. The future runner must implement and test product-specific pause/resume semantics without reordering the fixed queue.

- **AI-REPRO-01-M04 — OSF/arXiv artifacts are staged but not yet release-ready.** OSF administrator/license decisions, review archives, authentication verification, final commit replacement, timestamp, and identifier are explicitly pending (`research/llm-review-sequences/study.json:394-406`; `research/llm-review-sequences/preregistration-v2.md:174-182`). Add a documented manuscript build and source/PDF checksum record before publication packaging.

## 7. Required manuscript limitations

The final manuscript must explicitly retain or add these limitations:

- The zero-additional-spend design depends on pre-existing paid ChatGPT and Claude Max entitlements. It is not a free-to-reproduce design for researchers without equivalent accounts.
- Client version and requested alias do not uniquely identify provider-controlled routing, hidden instructions, or the served underlying model. Returned model identity may not be exposed sufficiently to prove model stability.
- Collection across subscription reset windows can span calendar time, creating platform-drift and time-order risk even with a randomized schedule. Collection timestamps, pauses, suspensions, and review-system timing balance must be reported.
- Product surfaces differ in default system prompts and tool affordances; results compare subscription-backed products, not isolated base models. The existing qualification at `research/llm-review-sequences/preregistration-v2.md:61-74` and `research/llm-review-sequences/paper/paper.tex:168-201` must remain.
- AI-assisted author-orchestrated internal review is conflicted and is not independent human validation. The existing statement at `research/llm-review-sequences/paper/paper.tex:302-306` must remain.
- The manuscript must not state that runner behavior or provenance capture was validated until an end-to-end no-benchmark preflight and fabricated-response integration test demonstrate the implemented path.
