## 4. Blocking findings

- **METH-B01 — The implemented confirmatory analysis cannot analyze the active design and retains forbidden legacy inference.** The active design contains only `local` context and one trial (`study.json:46-56`; `preregistration-v2.md:49,85`). `confirmatoryAnalysis` nevertheless requires both `local` and `cumulative` cells for H1 and workflow contrasts (`analyse.mjs:529-564`), causing the reproduced incomplete-cell failure. It also defaults to bootstrap seed `20260716`, rather than preregistered `20260718` (`analyse.mjs:338-340`; `preregistration-v2.md:125`), and returns a 90% workflow interval plus an `equivalent` decision (`analyse.mjs:374-404`) despite the plan requiring an exploratory 95% interval and explicitly prohibiting equivalence inference (`preregistration-v2.md:32-34,127-132`; `paper/paper.tex:236-241`). The CLI path only emits descriptive metrics and does not execute the confirmatory analysis (`analyse.mjs:615-622`). Freeze and test an active-design analysis path before collection.

- **METH-B02 — The stated missingness bounds are not valid bounds for H2.** The plan makes favourable and unfavourable bounds mandatory for primary inference (`preregistration-v2.md:134`). The implementation assigns missing blocks solely by intent (`analyse.mjs:480-490`) and then reports H2 robustness (`analyse.mjs:439-448`). When both malicious atomic and split cells are missing, both implemented assignments move them together, producing H2 zero under both “bounds”; valid H2 extrema require opposing assignments by decomposition. Implement estimand-specific bounds: malicious-versus-benign assignments for H1 and atomic-versus-split assignments for H2, with tests covering simultaneous missingness in both treatment arms.

## 5. Major findings

- **METH-M01 — Multiplicity handling is neither operationally specified nor implemented.** The preregistration only says secondary comparisons will be “Holm-adjusted within outcome families” (`preregistration-v2.md:136`) without defining those families, the adjusted quantity, or how adjusted inference is constructed from bootstrap distributions. `analyse.mjs:374-414` returns unadjusted percentile intervals and decisions; there is no Holm implementation or test. State whether the two primary directional claims are separate or a family, describe the implied alpha allocation, define every secondary family, and implement the prespecified adjustment.

- **METH-M02 — The active power audit omits H1.** H1 and H2 are both primary (`preregistration-v2.md:23-30`), but the active audit reports only split-effect power, workflow-equivalence assurance, and interaction power (`study.json:61-67`; `preregistration-v2.md:97-101`). The reproduced simulation used `runPowerSimulation`, which models malicious treatment cells but no matched benign outcome or H1 intent-discrimination calculation (`design.mjs:288-333`). Add a prospective H1 operating-characteristic or precision calculation, or explicitly state that no H1 power analysis was performed.

- **METH-M03 — The target population behind structural-template bootstrap inference is underdefined.** The sample is a deterministic five-index diagonal per family with one identifier instance (`preregistration-v2.md:51-55`), while inference resamples those five fixed templates as exchangeable units within each family (`preregistration-v2.md:123-125`; `analyse.mjs:345-370`). Define whether inference is conditional on these 40 templates or targets a superpopulation of templates. If the latter, justify the exchangeability claim. One lexical instance per template prevents separating structural-form variation from invented-identifier variation.

## 6. Minor findings

- **METH-m01 — Active and legacy fields remain easy to confuse.** `study.json:3-6` says the manifest is frozen, while the active nested design is still prospective (`study.json:21-25`) and top-level legacy factors and three-trial settings remain at `study.json:88-103`. The analysis currently follows those legacy context settings. Move legacy fields under an unmistakable namespace or require active-design configuration explicitly.

Align the simulation decision rule with the registered rule; the numerical difference under the stated negative alternative may be small but should not be assumed.

The manuscript notes only two products and limited external validity (`paper/paper.tex:285-294`) but should state directly that the systems were selected from subscriptions already held by the author.

## 7. Required manuscript limitations

The manuscript must retain its existing low-power and external-validity limitations and explicitly add:

- The review products are a convenience sample determined by the author’s existing subscriptions and US$0 additional-spend constraint; results do not generalize probabilistically to other products, API surfaces, or base models.
- Inference is conditional on 40 purposively selected diagonal templates unless a defensible template superpopulation is specified.
- One identifier instance per structural template confounds structural form with its lexical instantiation and provides no evidence about identifier-instance variability.
- One trial per cell prevents estimation of within-product stochastic consistency.
- H2 power is approximately `0.2584`, workflow-equivalence assurance `0.0038`, and workflow-interaction power `0.1016`; no H1 power calculation is currently supplied.
- Null results cannot establish no decomposition effect, workflow equivalence, or no interaction.
- Family-stratified percentile intervals based on five fixed templates per family may be coarse and depend on exchangeability within those small strata.
- Product-level effects combine hidden system instructions, routing, serving stack, authentication surface, and requested model; they are not isolated base-model effects.
