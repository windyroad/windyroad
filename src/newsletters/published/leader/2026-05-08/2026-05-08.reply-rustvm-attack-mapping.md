Both AISI-designed, not ATT&CK-mapped.

**rust_vm**: a reverse-engineering challenge. Stripped Rust ELF running a custom VM plus bytecode. Recover the instruction set, build a disassembler, reverse the authenticator, solve for a password. Built with Crystal Peak Security and Irregular.

**The Last Ones (TLO)**: a 32-step corporate-network range built with SpecterOps. Four subnets, ~20 hosts. Recon, credential theft, lateral movement across AD forests, CI/CD supply-chain pivot, exfiltration.

The article doesn't map TLO to ATT&CK. But every TLO step-type is an ATT&CK tactic, so the underlying behaviour is real-world even if the named tasks aren't ATT&CK-tagged.

The credibility test isn't "is this real?" but "did real red-team practitioners design it?". SpecterOps + Crystal Peak + Irregular: yes.
