## Analysis

### Differentiation

Shipped Product is supported by two high-visibility dependencies on the map: Engineering Team Capability and Delivery Pipeline. Both are what the reader's organisation pays for; everything lower on the map is substrate that feeds them. This matters because an AI Coding Assistants choice is effectively an Engineering Team Capability decision, and a Delivery Pipeline choice determines where AI Security Tooling hooks in. Read the custom components below with that chain in mind: what looks like deep-substrate tooling shapes the two top-visibility components the business depends on.

Agentic Engineering Practices sit at Genesis (0.15) with high visibility (0.78), AI Security Tooling sits at Genesis (0.14, shifted from 0.12 this week as OpenAI shipped its first named security products), Model Evaluation Frameworks at early Custom-Built (0.24, shifted from 0.22 this week on HealthBench Professional), AI Agent Runtimes (0.34, shifted from 0.32 this week on Workspace Agents), AI Regulation and Compliance (0.28, holding), Engineering Team Capability (0.25), and Prompt and Context Management (0.30, shifted from 0.28 this week) all sit in the Custom-Built band or below. Agentic Engineering Practices hang directly off Engineering Team Capability: how a team designs spec-driven workflows, agent skills, and feedback sensors is now a team-capability question, not a tool-choice question, which is why this Genesis-band component carries 0.78 visibility despite being newly-emerging on the evolution axis. These six are where durable advantage is still available to engineering leaders because none of them are buy-off-the-shelf yet. AI Agent Runtimes is a concentration risk: four downstream dependencies (Foundation Models, Model Hosting and Inference, Prompt and Context Management, Model Evaluation Frameworks) rely on it, so a runtime choice that cannot be reversed amplifies every other bet; this week's convergence on standards (OpenAI Agents SDK v2, NIST standards initiative, Spec-Kit frameworks) narrows the window to make that choice consciously. The investment that compounds fastest is Engineering Team Capability paired with Model Evaluation Frameworks: better evaluation creates the feedback loop that grows team capability, and capable teams build evaluation that targets the right failure modes; reassess this pairing when a vendor evaluation product covers the current use-case set for less than six months of internal development cost, or when quarterly evaluation-run volume falls below one per AI-assisted change. Prompt and Context Management and AI Agent Runtimes are worth revisiting quarterly as the ecosystem standardises. Thoughtworks Radar Vol 34 named "Codebase cognitive debt" as a Caution this quarter, which is the same class of risk as concentration on an unreversible agent-runtime pick: teams that adopt AI-generated patterns without understanding them compound a debt that only surfaces at maintenance. **What to do:** treat Engineering Team Capability and Model Evaluation Frameworks as the two budget lines that never lose funding this year; defer Prompt and Context Management build-out until the second standardisation signal lands.

### Risk

External vendor exposure: when a dominant coding-assistant vendor changes its default model with less than 30 days notice, or moves a previously-included model behind a higher pricing tier (the GitHub-restricting-Opus-4.7-to-premium pattern this week), integrations using vendor-default-model assumptions break within one release cycle, and the fallback is an abstraction layer in front of the assistant before consolidation arrives. Commodity layer: Model Hosting and Inference (Commodity, 0.70) could change terms; trigger is a hosting provider announcing a deprecation or a pricing change above 20%, affected components are AI Coding Assistants and AI Agent Runtimes, and the fallback is a multi-provider abstraction paired with a contract termination review. Regulatory shift: trigger is an AU, EU, or US regulator naming a concrete practice that must stop or start (example from this week: the US v. Heppner ruling means AI chat logs are now discoverable in US federal litigation, which changes retention and logging defaults for every engineering team doing AI-assisted customer or legal work), and without an inventory of AI-assisted code and AI chat interactions, the time to answer a regulator is measured in months not hours. Internal evaluation cadence: teams that pause model evaluation for AI-assisted changes see silent quality drift set in within weeks as the feedback loop stops producing signal, and the trigger to watch is any four-week stretch with no evaluation run; this is the only risk on the map engineering leaders directly control. Codebase cognitive debt (Thoughtworks Radar Vol 34 Caution this week): teams that adopt AI-generated code patterns without understanding them incur hidden maintenance debt that compounds, and the trigger is any pull request where the author cannot explain the pattern's tradeoffs on review. **What to do:** build the AI-assisted-code inventory this quarter (the regulatory lever means this is no longer optional), set an evaluation-run watchdog that alerts on four-week gaps, and add a "can you explain the pattern?" question to the pull-request template.

### Decisions

AI Evaluation Framework investment and AI Agent Runtime investment compete for the same engineering time, and this week's convergence on agent standards narrows the reassess window. Build a shared evaluation harness when three internal teams have asked for one in the last quarter, because evaluation feeds every other AI-assisted decision downstream; reassess in six months if a vendor product covers the full evaluation use-case set for less than six months of internal development cost. Adopt a managed Agent Runtime when either a second tier-1 model vendor has shipped a first-party runtime, or an international standards body, an open specification effort, and a widely-adopted agent framework have all reached the Trial-ring equivalent on three consecutive major-advisory editions; reassess if no new standards convergence signal lands inside a six-month window. If twelve months pass with no internal team requesting shared evaluation, revisit whether AI-assisted change is happening at a scale that justifies the framework. **What to do this quarter:** budget one engineer-quarter for the evaluation harness if the three-team trigger has fired; otherwise hold the resource and watch the second tier-1 first-party runtime as the runtime-lock-in signal that activates the Agent Runtime decision.

## How this map is used

The Engineering Leader persona (see `docs/JOBS_TO_BE_DONE.md`) is the reader this map serves: responsible for delivery pipeline, security posture, and team capability, stack-agnostic, and credential-sensitive.

1. The `/wr-newsletter` skill reads this file before drafting each weekly brief. The current landscape is the context the skill uses to decide which stories represent real movement versus surface noise, and to write the "map movement" line that appears on each brief item.
2. After drafting, the skill proposes updates to `ai-landscape.owm` for any component that moved this week, any new component that emerged, or any dependency that shifted. The update is captured in git so the week-to-week delta is visible.
3. Tom refines positions whenever he disagrees with the skill's placement. Edit `ai-landscape.owm` by hand, then re-render via the `wr-wardley:generate` skill (pointed at this OWM file) or by invoking its `owm-to-svg.mjs` converter directly. The skill command is the canonical path; the direct converter is an escape hatch for fast iteration.

- AI Regulation and Compliance sits at 0.60 visibility, which is high. Debatable: regulation is mostly invisible until it isn't. The week's regulatory movements (EU enforcement, US legal ruling, OAIC guidance) argue for keeping it visible; adjust downward if the run of regulatory news subsides.
- No evolve arrow on Compute and GPU Infrastructure or on the anchor-adjacent components. Compute is already commodity and does not need one; the anchor components are stable at their current positions.
- Source coverage in recent editions has been partial: OpenAI direct page consistently returns a 403 Cloudflare challenge (covered via Google News RSS site:openai.com workaround), Reddit was blocked by the tool chain (r/LocalLLaMA and r/MachineLearning lost), US FTC and OECD endpoints failed.
