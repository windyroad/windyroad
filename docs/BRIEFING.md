# Briefing

What a fresh session needs to know about this project beyond what CLAUDE.md, ADRs, JTBD, and the voice guide already cover.

## What You Need to Know

- **The Shift is Windy Road's weekly AI newsletter.** Pipeline: `/wr-newsletter` skill. Substrate: `docs/ai-engineering-brief/ai-landscape.owm` (Wardley map, internal only, never named in the reader body). Draft output: `src/newsletters/drafts/<persona>/YYYY-MM-DD.md`. Persona split: `leader/` (The Shift) and `developer/` (Tokens Spent). Published editions move to `src/newsletters/published/<persona>/`. Logo: `src/newsletters/assets/the-shift-logo.{svg,png,-300.png}` (done, reused every week).
- **Review gate order is critic -> voice -> content-risk** (ADR 016). Critic runs first because analytical quality is expensive to fix; voice and content-risk are cheaper polish. Each critic round is a separate subagent call (fresh context) to avoid confirmation bias. Max three rounds.
- **Per-item voice capture is how the opener is built.** SKILL step 10 uses `AskUserQuestion` per candidate (Agree / Adjust / Drop). Tom's Adjust text feeds the drafter; the "From Tom" opener is assembled from the strongest POV across the week's adjusts, not from a model guess. Capture transcript persists at `<draft-folder>/YYYY-MM-DD.capture.md` (ADR 019, lands 2026-04-25 P015 verifying); step 11 drafter rule preserves Tom's load-bearing noun-phrases verbatim.
- **Logo is reused.** The SVG is done. Do not rebuild the logo each week; only the draft body changes.
- **Every weekly issue gets a unique title.** "The Shift" is the series; individual editions each get a POV-carrying H1 (example: `Container wars again: agent runtimes just became default infrastructure`). Date goes in the subtitle line.
- **Link sources inline on claim text, not in a Source block.** LinkedIn renders inline Markdown links; a trailing URL block reads as a wall of raw text.
- **The first part of a new session is stale-marker territory.** Session-start hooks block edits until architect/JTBD/style/voice/a11y subagents run. Any new `.claude/agents/*.md` file is not discoverable until Claude Code session restart; fall back to `general-purpose` with the subagent's behaviour inlined until the next session.
- **Problem tickets live in `docs/problems/<state>/NNN-slug.md`** (per-state subdirectory layout, ADR-031: `open/`, `closed/`, `known-error/`, `verifying/`, `parked/`). Highest existing is 088; start at 089 for new tickets. (The old flat `docs/problems/NNN-slug.status.md` suffix layout was migrated; some briefing references and ticket bodies still cite the flat form.)
- **Stale-deps gate is governed by two ADRs in force together.** ADR-021 is the reactive bypass (`scripts/push-watch.sh` runs `npx dry-aged-deps --update --yes` pre-emptively before push and auto-commits if root manifests changed). ADR-022 is the proactive cadence (`.github/workflows/deps-refresh.yml` weekly cron on Mondays 09:00 UTC, opens a PR via `peter-evans/create-pull-request@v7` when manifests change). Both are intentional: cadence prevents drift accumulation; push-watch absorbs whatever drift accumulated since the last cadence run. Removing either leaves a gap. Both scope to root manifests only because the project is `private: true` (no published API surface, no changeset required).
- **Wrapper-as-defence-in-depth is the documented windyroad-local pattern for closing upstream-plugin gate gaps.** When an upstream `wr-*` plugin hook has a known gap (and the upstream issue is open with no movement), extend `scripts/push-watch.sh` and / or `scripts/release-watch.sh` with a project-local check that runs before the underlying tool fires. Bypass markers live in a wrapper-owned `/tmp/wr-push-watch-${CLAUDE_SESSION_ID:-no-session}/` directory, NOT the upstream plugin's `/tmp/risk-${SESSION_ID}/` namespace (layering violation). Examples: ADR-021 (dry-aged-deps reactive bypass + auto-commit) and ADR-028 (CI-status check via `scripts/ci-status-check.sh`, single-shot `red-ci-acknowledged` marker). Retire the wrapper check when upstream lands per the ADR's reassessment criteria.
- **Drafter is an inline main-assistant pass, not a subagent.** SKILL.md step 11 has no Agent invocation; the inline drafter reads SKILL.md as its working instructions. Voice / content-risk / SW-critic gates are subagents; the drafter is not. ADR 019 documents the rationale (P015 was a discipline failure, not a confirmation-bias failure; subagent promotion would lose AskUserQuestion conversation history).
- **Grep brand assets before designing new visuals.** `src/newsletters/assets/` holds the real Shift logo (SVG + PNG, 6-speed shift-gate icon + Futura wordmark). `public/img/` holds the site logo and the OG-image generation pattern (`scripts/generate-og-image.mjs`). Skipping the grep produced two image-design misfires this session (homemade wordmark, then stamped-as-block) before the right composition emerged.
- **`wr-risk-scorer:assess-release` SKILL.md step 5 has a contract violation.** Prose says "via the Skill tool" but provides `subagent_type:` (an Agent-tool parameter). For direct scoring use Agent tool with `subagent_type: wr-risk-scorer:pipeline`; for marker-write (push gate satisfaction) use Skill tool with `skill: wr-risk-scorer:assess-release`. Direct Agent invocation produces correct scores but does NOT write the bypass marker the push gate looks for. Reported upstream as windyroad/agent-plugins#110 (P047). Until the upstream fix lands, prefer the assess-release Skill when the marker is needed.
- **Risk scorer's release-layer doesn't auto-verify whether changeset commits are pushed.** It reads `.changeset/*.md` content and treats it as forward-looking "would deploy" risk. If the underlying feat/fix commits are already on origin/master (e.g. waiting for the changesets-action release PR to merge), the scorer will inflate release risk and trip ADR-042's above-appetite branch incorrectly. Workaround: supply commit-state context manually (use `git log --diff-filter=A` to identify the introducing commit, then `git merge-base --is-ancestor <commit> origin/master`) and override the score. Tracked as P046.
- **`scripts/push-watch.sh` auto-resolve updates package.json but not package-lock.json.** When the dry-aged-deps pre-push gate trips, push-watch.sh runs `npx dry-aged-deps --update --yes`, commits the package.json change as `chore(deps): refresh stale dependencies (P026)`, then retries push. The retry FAILS because the lockfile is still stale. Workaround: after the auto-resolve commit, run `npm install` manually, commit the lockfile sync as a follow-on, then re-push. The proper fix is a windyroad-local update-stale-deps skill (P026 is the tracking ticket; placement reframed mid-session as windyroad-local, not upstream wr-itil).
- **AFK `work-problems` re-selects tickets that are blocked on an out-of-iter resolution.** The orchestrator's WSJF ranking has no durable demotion for a ticket whose only remaining work needs someone outside the iteration: a user direction-decision (example: P065, the cog-a11y leader reading-level-target choice, routed to Tom 2026-06-02 and still unanswered) or an upstream maintainer (the placement-authority class P069 tracks). Such tickets cycle back to the top of the queue and each iter pays the read-and-skip cost to re-discover the block. Before spending an iter on a high-WSJF ticket, scan its latest Iter Notes for a "routed to ... / outstanding_questions" marker or an upstream-blocked classification; if present, skip without re-deriving. The durable fix (a blocked-and-not-self-resolvable demotion signal in the ranking) generalises P069 from placement-authority to any out-of-iter blocker.
- **`claude -p` subprocess quota is a separate pool from main session quota.** When an iteration subprocess returns "You've hit your limit · resets <time>" with a non-zero exit, that's the per-account hourly subprocess pool. The main session can continue working; just don't dispatch another `claude -p` until the reset time. Per work-problems SKILL.md Step 5 "Quota as the natural stop", quota-exhaust is the natural halt condition for AFK loops; orchestrator stops cleanly, summary emits with the partial-iter cost-metadata.
- **Ticket-family completeness check before declaring friction "done".** Closing the ticket you opened does NOT mean the friction class is resolved. Source-fetch friction (P010 + P014 both `.known-error.md`) recurred this edition: OpenAI 403 needed Google News fallback, news.google.com redirect URLs needed Playwright, Verge/WaPo got rate-limited. Each edition we re-derive the same workaround. Same shape on cover-image (P037 + P044 still open while P011 is verifying). Before marking a ticket done, scan sibling tickets in the same friction class. If any are open or known-error and the friction recurs, the friction is NOT done; the closure is bookkeeping, not resolution. (2026-05-08 marquee retro finding.)
- **Pipeline-meta is banned from reader-facing body.** No "(URL not Playwright-resolvable due to Cloudflare; available via search at outlet.com)" or similar diagnostic prose in the brief body. P036's editorial-meta rule covers evidence-stance language; this is a sibling case the rule did not catch. If the pipeline can't resolve a source, ASK Tom for the URL via `AskUserQuestion` rather than writing the limitation into the body. Recurrence on 2026-05-08, Tom called FFS and re-stated explicitly.
- **Playwright Google News URL resolver script lives at `scripts/resolve-gnews-urls.mjs`.** Headless Chromium navigates `news.google.com/rss/articles/...` redirect URLs and extracts the underlying outlet article URL. Hardcoded URL list per-edition, pattern reusable. Used during 2026-05-08 finalise when WebFetch was rate-limited. Closes a piece of P014's tool-blocked-source helper need; the helper proper (a wr-newsletter-integrated step that fans out the resolver against the source-failures list) is still wanted.

## What Will Surprise You

- **Tom's editorial reviews repeatedly catch weaknesses the critic subagent does not.** Redundant preamble, author-voice authenticity, subtle reader-respect patterns, LinkedIn-format issues, claim-evidence looseness. The structural rubric is tight on correctness and loose on presentation and persuasion. P008 option 2 (editorial-sim subagent) shipped 2026-04-25 as ADR 020 + `.claude/agents/wr-newsletter-editor.md`: SKILL.md step 15.25 invokes the agent between sw-critic and LinkedIn-post drafting, scoring would-open / would-read-through / would-forward against the persona's documented JTBD context. Verification fires on next `/wr-newsletter` run. Options 1 (rubric expansion 21-31) and 3 (content-risk subagent per ADR 018) shipped earlier; all three options now in flight.
- **The critic subagent defined at `.claude/agents/wr-sw-critic.md` will not be discoverable in the session it was created in.** Work around with `general-purpose` plus the critic process inlined. See problem 007.
- **The `@windyroad` plugins can be silently disabled by an uncommitted `.claude/settings.json` edit, which degrades the newsletter pipeline.** On the 2026-05-25 session the `enabledPlugins` block had been stripped to just `skill-creator`; the `wr-voice-tone` gate was unavailable and the run fell back to a `general-purpose` subagent reading `docs/VOICE-AND-TONE.md` directly (it worked, but missed the dedicated agent's stricter reader-respect checks). Reverting `settings.json` does NOT restore plugins mid-session: plugin/agent registration happens at session start, so a restart is required before the agents reappear. Sibling to the new-agent-discoverability point in "What You Need to Know".
- **Content-risk review now runs as a fresh-context subagent** via `wr-content-risk-scorer` (ADR 018, lands 2026-04-25). Same five-axis output format as the inline path (pinned by ADR 015 confirmation criterion 3); the only change is the execution context shifts from inline to fresh subagent. See ticket 009 (verification pending; close on next live `/wr-newsletter` exercise).
- **OpenAI's news page returns 403 to `WebFetch`** and 401 to `curl`-with-browser-UA (Cloudflare challenge). Proven workaround this session: Playwright via `node -e "import('playwright')..."` with Chromium UA passes the challenge and returns 200; openai.com slug convention is `openai.com/index/<slug>/` and slugs follow the post title closely (verify-and-iterate works). Google News RSS scoped `site:openai.com` is a lighter-weight fallback for titles only. Reuters is 401-blocked even via Playwright; no workaround found this session. Reddit is blocked at the Claude Code tool layer entirely. See problems 010 and 014.
- **Visual artifacts (SVG logos) need render-and-verify discipline**. `sips -s format png -Z <size> in.svg --out out.png` + `Read` on the PNG. Tom had to direct this explicitly after multiple iterations shipped unverified. See problem 011.
- **The Wardley map is internal-only substrate**. Never named, never referenced in body copy. The reader sees the analysis that results from the map, not the map vocabulary (no "map", "landscape", "position", "evolution axis", "commodity", "genesis" in reader-facing prose).
- **Map anchor is a preference, not a hard filter.** Three-lens-strong items without a clean map mapping still qualify. The original "every item must anchor" rule was softened.
- **Tier-1 source rule is a heuristic, not a hard block.** If one tier-1 source fails but tier-2 coverage is rich, judgement says proceed with the map update and note the partial coverage. Blind rule-following produced a bad call earlier in the session.
- **Publication-day workflow is multi-hour under current pipeline.** The `/wr-newsletter` skill runs end-to-end in one sitting (sources, map, critic loops, image, LinkedIn post, publish). Friday-morning sessions regularly take 2 to 4 hours. Phase split into prep (Mon to Thu) plus finalise (Fri) is planned. See P018.
- **The filter step drops significant stories on attribution heuristic without attempting corroboration.** The Apple-CEO transition was missed on this pattern in the 2026-04-24 edition; multi-outlet corroboration via Google News was trivially available. Filter-step corroboration branch planned. See P016.
- **The critic rubric did not catch internal contradictions, referential ambiguity, jargon density, voice drift, scaffolded headlines, news-peg-first H1s, or unsupported author commitments.** P017 third-wave rubric expansion shipped 2026-04-25 (verifying): newsletter-critic-rubric.md grew from 31 to 38 checks (added check_32 internal-consistency, check_33 referential-clarity, check_34 density / consultant-speak, check_35 voice consistency across sections, check_36 item-headline distinctiveness, check_37 headline thesis-first, check_38 author-commitment honor-ability). - **WSJF formula = `(Severity × Status Multiplier) / Effort` per upstream wr-itil:manage-problem.** Status Multiplier is 2.0 for Known Error, 1.0 for Open, 0 for Verification Pending and Parked. - **Cross-edition thesis contradiction is a real failure mode.** No automated check exists for consistency between consecutive editions. Issue 07's prep brief Item 1 human angle directly contradicted Issue 06's H1 thesis ("AI's real prize is capacity, not a smaller team"). Tracked as P080.

- **The architect Decisions Compendium cannot be refreshed compliantly in this repo right now.** `wr-architect-generate-decisions-compendium` (wr-architect v0.16.0) emits U+2014 em-dash heading separators (in `### ADR-NNN` title lines) that this project's P025 no-em-dash Bash hook blocks, while the `architect-compendium-refresh-discipline.sh` PreToolUse hook denies any ADR-changing commit whose staged compendium does NOT match generator output. That is a deadlock: regenerate and the em-dash hook denies; stay compliant and the discipline hook denies. The documented escape is the commit-message token `RISK_BYPASS: architect-compendium-deferred` (the discipline hook short-circuits on it). When you flip or edit an ADR, expect to carry that token and defer the compendium refresh. Tracked as P087.
- **The architect edit-gate hits a session-ID race under AFK subprocesses (P260/ADR-050 class).** Delegating to `wr-architect:agent` writes the `/tmp/architect-reviewed-<sid>` marker under the SUBAGENT's observed session_id, but your main-iter Edit hook may read a different runtime session_id and deny with "No architect review marker found for this session". The itil create-gate solved this with ADR-050 Option C (candidate-SID marker write); the architect gate has not adopted it. Workaround once the review has genuinely run: `touch /tmp/architect-reviewed-<sid>` for each recent UUID in `/tmp/itil-runtime-sid-tomhoward-*.current` (omit the `.hash` sibling so the no-hash allow path is taken and successive ADR edits in one reviewed change set do not trip inter-edit drift). Tracked as P088.

## Cross-project patterns (from 86-session insights report, 2026-03-17 to 2026-04-16)

- **Risk-scorer above appetite is a hard stop, not an advisory signal.** Across projects, Claude attempted releases despite the scorer flagging above appetite and was pushed back by the user in at least three sessions. On windyroad, enforce the same discipline on `git push`, `npm publish`, and any deploy command. See problem 012.
- **External-facing text (GitHub comments, LinkedIn teasers, PR bodies, release notes) needs a voice/tone check before posting.** AI tells have leaked into external comms and produced user frustration. The newsletter body now runs voice via `wr-voice-tone:agent`; teaser copy, CTAs, and outbound GitHub text should too. See problem 013.
- **Output token limits bite on long sessions.** Checkpoint with commits frequently (after each logical batch, not at the end), and prefer short status lines over verbose narration between edits. Losing a session's final publish step to a token cutoff is a recurring pattern.
- **TDD RED-state verification matters.** Confirm the test fails for the right reason (matches the bug signature, not unrelated setup) before writing the fix. Grep-based BATS assertions in particular are brittle after renames and produce false positives.
- **Read all inputs before analysing, do not sample.** BRIEFING.md, risk reports, user screenshots, referenced ADRs: read them in full on first contact. Shallow sampling has misdiagnosed a live production outage on another project and can bite the same way on windyroad when multiple docs govern a change.

## Gate + dependency friction (2026-06-15 AFK work-problems session)

- **The commit gate discards working-tree edits when it blocks.** When the external-comms / risk commit gate denies a `git commit`, the working-tree changes staged for that commit are silently reverted, not just left unstaged. Observed appending resolution notes to two problem tickets: each blocked retry lost the appends, so re-appending produced a new staged-content hash that re-invalidated the P085 marker, looping. Workaround that AVOIDS the loop: pre-set the gate markers by running `wr-risk-scorer:external-comms` + `wr-voice-tone:external-comms` (and `wr-risk-scorer:pipeline` for the commit gate) on the EXACT final commit message BEFORE the first `git commit` attempt, so the commit never blocks and nothing is discarded. This is more severe than P085 (data loss vs re-review friction) and may warrant its own ticket.
- **dry-aged-deps can demand a version that `--update` will not apply when a dep is pinned exact.** playwright was pinned to exact `1.59.1`; `dry-aged-deps --check` flagged `1.60.0` as overdue (33d >= dev minAge 30) and blocked the push, but `dry-aged-deps --update --yes` would not bump it because the exact pin caps the semver range below 1.60.0. Resolution per P026 discipline: run the upgrade explicitly (`npm install --save-dev --save-exact playwright@1.60.0`), not add an exclusion. Exact-pinned deps with no `.dry-aged-deps.json` exclusion reason are upgrade targets, not holds.

## Newsletter pipeline gates vs structural defects (2026-06-15 Issue 09 retro)

- **The five newsletter gates (voice, content-risk, critic, editor, cog-a11y) + cross-edition + URL-verify can all PASS on a draft that still has structural and sourcing defects.** Issue 09 passed every gate first-time yet Tom flagged 8 defects: redundant Source lines when inline links exist, outlets named without linking, over-proving a first-hand/widely-known story, dropped "Also worth noting" section, missing "Issue NN:" H1 prefix, missing CTA horizontal rule, inconsistent model naming across brief vs LinkedIn, and sparse dated citations. The gates are LLM-judgement reviewers; none checks deterministic format invariants. Captured as P089 (proposed fix: deterministic structural-lint check). "Rubric PASS does not mean the artefact is good" recurs on the structural axis.
- **The newsletter CTA must NOT carry the Windy Road services pitch.** The CTA is the rotating reader-invitation + linked windyroad.com.au only. The leader persona-config "Description variants" are the recurring source of the pitch; the drafter pulls one every edition and Tom removes it every edition ("I've said that before"). Captured as P090.
- **When the pipeline cannot resolve a source URL, ASK Tom rather than dropping the outlet name or naming-without-link.** Google News RSS only exposes bare publisher domains; paywalled outlets (WSJ/Bloomberg/NYT/Axios) 403 on curl. Tom is willing to supply canonical URLs on request ("If you cannot resolve a URL, ask me"). Do not silently degrade attribution. Captured as P091.
- **push:watch is hostile to the amend-to-fix-a-just-pushed-commit workflow.** Its auto pull-rebase replays a sibling-amend onto origin and injects conflict markers into working files mid-session. Recover via `git rebase --abort` + `git push --force-with-lease`. push:watch also false-fails ("Pipeline failed") on transient GitHub API network errors (connection reset) when CI actually succeeded; verify the real conclusion via `gh run view <id> --json conclusion` before believing the failure. Captured as P092.
- **Pre-set the commit-gate markers before committing to avoid the P085 discard loop.** The external-comms commit gate discards working-tree edits when it blocks, and re-staging changes the marker hash, looping. Run `wr-risk-scorer:pipeline` + `wr-risk-scorer:external-comms` + `wr-voice-tone:external-comms` on the exact final commit message BEFORE the first `git commit` so the commit never blocks.
